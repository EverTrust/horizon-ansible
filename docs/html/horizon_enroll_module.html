evertrust.horizon.horizon_enroll – Horizon enrollment plugin

Note

This plugin is part of the evertrust.horizon collection (version 0.1.1).

To install it use: ansible-galaxy collection install evertrust.horizon.

To use it in a playbook, specify: evertrust.horizon.horizon_enroll.

Synopsis

  • Performs an enrollment against the Horizon API.

Note

This module has a corresponding action plugin.

Requirements

The below requirements are needed on the host that executes this module.

  • cryptography>=3.4.0

Parameters

Parameter Choices/Defaults Comments
ca_bundle
string
The location of a CA Bundle to use when validating SSL certificates.
client_cert
string
The location of a client side certificate.
Required if you use certificate authentication
client_key
string
The location of a client side certificate's key.
Required if you use certificate authentication
endpoint
string / required
Horizon installation endpoint
Should include the protocol (https://) and no trailing slash
key_type
string / required
Type of key to encode
labels
dictionary
Labels of the certificate
mode
string
    Choices:
  • centralized
  • decentralized
enrollement mode
password
string
Security password for the certificate.
Password policies will be applied to check validity.
Required only if the enrollement is centralized and the password generation mode is not random.
profile
string / required
Name of the profile that will be used to enroll the certificate.
sans
dictionary / required
Subject alternative names of the certificate
subject
dictionary / required
Certificate subject.
You can either give the description of the subject, or the full dn.
If you give the dn, other values won't be used.
x_api_id
string
Horizon identifier
Required if you use password authentication
x_api_key
string
Horizon password
Required if you use password authentication

Notes

Note

  • Enrolling a certificate requires permissions on the related profile.

  • Be sure to use the “Enroll API” permission instead of “Enroll”.

Examples

- name: Simple centralized enrollment
  evertrust.horizon.horizon_enroll:
    # login and password to connect to the API
    endpoint: "https://<api-endpoint>"
    x_api_id: "<horizon-id>"
    x_api_key: "<horizon-password>"
    mode: "centralized"
    password: "pAssw0rd"
    key_type: "rsa-2048"
    profile: "profile"
    subject:
      cn.1: "myCN"
    sans:
      dnsname.1: "myDnsname"
    labels:
      snow_id: "value1"
      exp_tech: "value2"
- name: decentralized enrollment with csr
  evertrust.horizon.horizon_enroll:
    # login and password to connect to the API
    endpoint: "https://<api-endpoint>"
    x_api_id: "<horizon-id>"
    x_api_key: "<horizon-password>"
    mode: "decentralized"
    csr: <a_csr_file>
    password: "pAssw0rd"
    key_type: "rsa-2048"
    profile: "profile"
    subject:
      cn.1: "myCN"
      ou.1: "myFirstOU"
      ou.2: "mySecondOU"
    sans:
      dnsname:
        - "myDnsName1"
        - "myDnsName2"
    labels:
      snow_id: "value1"
      exp_tech: "value2"
- name: decentralized enrollment without csr
  evertrust.horizon.horizon_enroll:
    # login and password to connect to the API
    endpoint: "https://<api-endpoint>"
    x_api_id: "<horizon-id>"
    x_api_key: "<horizon-password>"
    mode: "decentralized"
    password: "pAssw0rd"
    key_type: "rsa-2048"
    profile: "profile"
    subject:
      cn.1: "myCN"
      ou:
        - "myFirstOU"
        - "mySecondOU"
    sans:
      dnsname.1: "myDnsname"
    labels:
      snow_id: "value1"
      exp_tech: "value2"
