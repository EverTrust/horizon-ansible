evertrust.horizon.horizon_feed – Horizon feed plugin

Note

This plugin is part of the evertrust.horizon collection (version 0.1.1).

To install it use: ansible-galaxy collection install evertrust.horizon.

To use it in a playbook, specify: evertrust.horizon.horizon_feed.

  • Synopsis

  • Parameters

  • Notes

  • Examples

Synopsis

  • Present a certificate from a discovery campaign to be used by Horizon.

Note

This module has a corresponding action plugin.

Parameters

Parameter Choices/Defaults Comments
ca_bundle
string
The location of a CA Bundle to use when validating SSL certificates.
campaign
string / required
Name of the discovery campaign.
certificate
string
A certificate pem, or the path to the certificate pem file.
client_cert
string
The location of a client side certificate.
Required if you use certificate authentication
client_key
string
The location of a client side certificate's key.
Required if you use certificate authentication
endpoint
string / required
Horizon installation endpoint
Should include the protocol (https://) and no trailing slash
hostnames
list / elements=string
Hostname of the discovered host.
ip
string / required
IP adress
operating_systems
list / elements=string
Operating system of the discovered host.
paths
string
Path where the certificate was discovered.
usages
string
Free field usually used to indicate configuration files making use of the certificate.
x_api_id
string
Horizon identifier
Required if you use password authentication
x_api_key
string
Horizon password
Required if you use password authentication

Notes

Note

  • Feeding a certificate requires permissions on the specified discovery campaign.

Examples

- name: Test discovery
    evertrust.horizon.horizon_feed:
      endpoint: "https://<api-endpoint>"
      x_api_id: "<horizon-id>"
      x_api_key: "<horizon-password>"
      campaign: campaign1
      ip: localhost
      certificate: <certificate_in_pem>

- name: Test discovery
    evertrust.horizon.horizon_feed:
      endpoint: "https://<api-endpoint>"
      x_api_id: "<horizon-id>"
      x_api_key: "<horizon-password>"
      campaign: campaign1
      ip: localhost
      certificate:
        src: pem/file/path
